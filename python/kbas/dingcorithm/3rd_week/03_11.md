# Stack vs Queue vs Hash Table

| 경우             | Stack                                                 | Queue                                     | Hash Table                                                                                                                                                                   |
| ---------------- | ----------------------------------------------------- | ----------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 특정 원소 조회   | O(N)                                                  | O(N)                                      | **Ω(1)** / O(N)                                                                                                                                                              |
| 중간에 삽입 삭제 | O(N)                                                  | O(N)                                      | **Ω(1)** / O(N)                                                                                                                                                              |
| 데이터 추가      | O(1)                                                  | O(1)                                      | **Ω(1)** / O(N)                                                                                                                                                              |
| 범위 탐색        |                                                       |                                           | 이 경우에는 해쉬 테이블이 더 안 좋음                                                                                                                                         |
| 정리             | 가장 최근에 들어온 순서대로 처리되어야 하는 일에 필요 | 들어온 순서대로 처리되어야 하는 일에 필요 | 해시 충돌이 잘 안 일어날 수 있는 상황에서는 매우 좋다. 따라서 해시 함수가 잘 데이터를 흩뿌러줄 있는 걸 골라야 한다. 현업에서는 성능이 안 겹칠 것이라는 가정을 애초에 갖고 감 |

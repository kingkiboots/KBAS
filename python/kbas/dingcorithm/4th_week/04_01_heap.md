# 힙

> 힙은 데이터에서 최대값과 최소값을 빠르게 찾기 위해 고안된 완전 이진 트리

새로운 우선 순위가 들어올 때마다 순서를 정렬해준다.

정렬이랑 무슨 차이인가?
정렬은 NlogN이고,
힙은 데이터를 가져오고 뺴는 데 log(N) 만큼의 시간 복잡도가 걸리는 대신 매번 정렬을 해주지 않아도 정렬된 상태를 유지할수 있다.

그러니 항상 최대 혹은 최소의 값들이 필요한 연산이 있다면 힙을 사용하면 된다.

힙은 항상 큰 값이 상위 레벨에 있고 작은 값이 하위 레베에 있도록 하는 자료구조.
다시 말해 부모 노드의 값이 자식 노드의 값보다 항상 커야한다.
가장 큰 값을 가장 커야하므로 가장 위로 간다.
그래서 최대의 값을 빠르게 구할 수 있는 것이다.

힙도 두종류가 있다.

- Min Heap: 부모가 자식보다 작음
- Max Heap: 부모가 자식보다 큼

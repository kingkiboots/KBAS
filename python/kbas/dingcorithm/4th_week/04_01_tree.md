# 완전 이진 트리

완전 이진 트리를 코드로 만들어보자

```
       1
   2       3
 4   5   6
```

[None, 1, 2, 3, 4, 5, 6]

0 번 인덱스는 None으로 비워준다.

다음과 같은 방법으로 트리 구조를 파악할 수 있다.

1. 현재 인덱스 \* 2 -> 왼쪽 자식의 인덱스
2. 현재 인덱스 \* 2 + 1 -> 오른쪽 자식의 인덱스
3. 현재 인덱스 // 2 -> 부모의 인덱스

이렇게 배열을 이용하면 트리구조라는 비선형적 자료구조임에도 불구하고 쉽게 데이터를 드러낼 수 있음
만일 이 방법을 사용하지 않는다면 타고 타고 들어가야겠지. 그리고 배열을 통해 트리에 있는 노드간의 관계를 쉽게 찾아갈 수 있다.

이게 왜 됨??
level0 : 1개
level1 : 2개
level2 : 4개
level3 : 8개
...
levelk : 2^k개

그럼 여기서 퀴즈
만약 높이가 h인데 노드가 꽉 차 있는 완전 이진 트리라면 모든 노드의 개수를 몇개 일까요?
--> 1 + 2^1 + 2^2 + 2^3 + 2^4 + ... + 2^h
즉 2^(h+1) - 1

반대로 노드의 개수가 N일 때 h는 몇일까요?
2(h+1) - 1 = N
h = log2(N+1) - 1

시간복잡도로 환산하면 대충 O(logN)이구나 라고 생각하면 된다.

> 언제 트리를 쓰면 좋은지를 아직 모른다. 그걸 알기 위해서는 Heap 이라는 자료 구조를 알아야 한다.

완전 이진 트리는 왼쪽부터 꽉찬다.

그래서 아래가 완전 이진 트리이다.

```
       8
    6      3
  4   2  5
```

## 완전 이진 트리 구현 방법

완전 이진 트리를 코드로 작성하는 방법은 배열을 사용하는 것이다.

```
        8        level 0 -> [None, 8] 첫번째 레벨의 8을 넣고
    6       3    level 1 -> [None, 8, 6, 3] 다음 레벨의 6, 3을 넣고
  4   2   5      level 2 -> [None, 8, 6, 3, 4, 2, 5] 다음 레벨의 4, 2, 5를 넣으면 된다.
```

```
  0    1  2  3  4  5  6
[None, 8, 6, 3, 4, 2, 5]
```

이 구조를 분석해보면 이런 법칙이 있다.

1. 현재 인덱스 \* 2 -> 왼쪽 자식의 인덱스
2. 현재 인덱스 \* 2 + 1 -> 오른쪽 자식의 인덱스
3. 현재 인덱스 // 2 -> 부모의 인덱스

한번 계산 해보자.

이런식으로 부모와 자식의 관계를 배열 속에서 완전 이진 트리를 나타내는 방법으로 쉽게 사용할 수 있는 방법이 배열!

## 완전 이진 트리 높이 구하는 방법

아래 그림과 같이 보면 각 레벨에 최대로 들어갈 수 있는 노드의 개수는 `2^k`다

```
       1          level 0 -> 1개
    2      3      level 1 -> 2개
  4   5  6   7    level 2 -> 4개
 8 9 ..       15  level 3 -> 8개
                  level k -> 2^k개
```

> 높이가 h인데 모든 노드가 꽉 차 있는 완전 이진 트리라면 모든 노드의 개수는 몇일까?

```
1 + 2^1 + 2^2 + 2^3 + ... + 2^h
즉 이를 수식으로 표현하면 2^(h+1) - 1 (공비수열의 합계)
```

> 반대로 최대 노드의 개수가 N일때 h는 뭘까

```
2^(h+1) + 1 = N
h = log_2(N+1)-1

그러니 완전 이진 트리의 노드 개수가 N일 때 최대 높이가 log2(n+1)-1 이니,
이진 트리의 높이는 최대로 해봤다 대충 O(log(N)) 이라고 생각하면 된다.
```
